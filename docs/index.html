SEED 情绪识别：ML 到 DL

本站点使用你已有的 EEG 特征（`data/EEG_features`）完成情绪识别基线，并把结果做成“报告 + 交互演示”的在线页面。

如何使用这个网站（新手友好版）

  1. 打开 交互：
    • 如果你想“做预测”：上传你生成的特征 `.mat`（与 `data/EEG_features` 同结构），选择特征类型（`de_lds / de / psd`），然后点击“开始分析”。
    • 如果你想“看基线”：不上传文件，使用页面下方的协议/指标下拉框，查看默认模型在两种评估协议下的汇总结果和平均混淆矩阵。
  2. 打开 报告：
    • 如果你想“更像实验报告”：上传 `.mat` 特征文件，点击“生成报告”，页面会给出该文件对应的预测分布、置信度统计以及解释说明。
    • 如果你不上传文件：页面会展示默认基线报告（两种协议的结果表格 + 静态图表），适合直接写作业/论文。
  3. 如果你只是要看“图”：看说明页里的“静态图表导出”模块（图片可直接用于写报告）。

流程说明（数据到网页的链路）

  1. 从 data/EEG_features 和 data/save_info 自动构建 trial 清单。
  2. 先跑 ML 基线（LR / SVM / RF），再跑 DL 基线（MLP）。
  3. 训练结束后导出 `docs/assets/results.json`，并生成 `docs/assets/figures/*` 静态图。

上传文件注意事项（非常重要）

  • 网页端上传/推理仅使用“特征级 `.mat` 文件”，不包含原始 EEG 也不进行滤波/去伪迹。
  • 上传的 `.mat` 需要与本项目特征命名规则一致，例如 `de_lds_1` / `de_lds_2` ... / `de_lds_80` 这类 key（或对应的 `de_`、`psd_` 前缀）。
  • 由于上传 `.mat` 本身通常不自带真实标签，因此上传报告默认不计算 Accuracy / Macro-F1；如果你提供真实标签（在本地计算后），下一步可以把指标也展示到网页。
