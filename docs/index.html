SEED 情绪识别：ML 到 DL

本站点使用你已有的 EEG 特征（`data/EEG_features`）完成情绪识别基线，并把结果做成“报告 + 交互演示”的在线页面。

如何使用这个网站（新手友好版）

  1. 打开 交互：
    • 如果你想“做上传子集分析”：上传一个 zip，目录必须是 `EEF_features/` 与 `save_info/`。
    • zip 内文件命名：`EEF_features/<subject>.mat` 与 `save_info/<subject>_<date>_<session>_save_info.csv`。
    • 建议每个被试都放 4 个 session 的 `save_info`（1..4），这样 trial 覆盖更完整（1..80）。
    • 如果你想“看默认基线”：不上传文件，使用页面下方的协议/指标下拉框查看已有结果。
  2. 打开 报告：
    • 上传 zip 后：页面会给出 subject_dependent 与 subject_independent 两协议的 Accuracy/Macro-F1。
    • 上传缺失时：按“有效覆盖样本”计算，并显示 valid/expected 覆盖率。
    • 不上传时：页面展示默认基线报告（两种协议表格 + 静态图）。
  3. 如果你只是要看“图”：看说明页里的“静态图表导出”模块（图片可直接用于写报告）。

流程说明（数据到网页的链路）

  1. 从 data/EEG_features 和 data/save_info 自动构建 trial 清单。
  2. 先跑 ML 基线（LR / SVM / RF），再跑 DL 基线（MLP）。
  3. 训练结束后导出 `docs/assets/results.json`，并生成 `docs/assets/figures/*` 静态图。

上传文件注意事项（非常重要）

  • zip 结构必须固定：`EEF_features/` + `save_info/`（目录名区分大小写）。
  • `EEF_features/*.mat` 需包含 `de_LDS_1..80`（或 `de_1..80` / `psd_1..80`）这类特征 key。
  • `save_info/*.csv` 每个 session 固定 20 行；情绪从第2列电影路径倒数第二段提取（如 `...\\happy\\xxx.mp4`）。
  • 首次上传时会加载 Pyodide 和 JSZip，等待时间会比后续更长。
