使用说明

本站点用于 SEED 情绪识别的上传分析与实验结果展示，包含报告页面与交互页面两类功能。

概念解析

de_lds 的含义

de_lds 是当前项目中最核心的特征之一。它可以理解为“经过时间平滑后的微分熵（Differential Entropy）”特征：先从每个频带提取微分熵，再通过类似 LDS 的平滑方式减少瞬时噪声，使得 trial 级特征在时间上更稳定。在 EEG 情绪识别中，这类平滑处理通常能让模型更容易学习到稳定的情绪模式。

de 的含义

de 指微分熵特征，是 EEG 频带能量分布的一种常见统计表达。与原始波形相比，de 更接近“信息量/不确定性”的表示方式，常用于把高维时序信号压缩成可用于分类的结构化特征。 在本项目中，de 通常与 trial、频带、通道三个维度共同组织。

pas 的含义

你这里写的 pas 在 EEG 特征语境里通常对应 PSD（Power Spectral Density，功率谱密度）。PSD 描述不同频率成分上的能量分布，是最经典也最直观的频域特征之一。 当你在页面中选择 psd 特征类型时，本质上就是让模型基于频带功率信息进行情绪分类。

logistic_regression 的介绍

Logistic Regression 是一个线性分类器，参数可解释性强、训练速度快，常作为情绪识别实验的基线模型。它对特征尺度较敏感，因此通常与标准化一起使用。 在本项目里，它的作用是提供一个“简单但稳定”的参照，帮助你判断更复杂模型是否真的带来增益。

random_forest 的介绍

Random Forest 是由多棵决策树组成的集成模型，擅长处理非线性关系，对特征分布的鲁棒性较好。它往往能在较少调参的情况下得到可用结果，但在跨被试泛化上不一定总优于线性模型。 在你的报告中，它更适合作为“非线性传统机器学习”代表进行比较。

svm_rbf 的介绍

SVM（RBF 核）通过核函数在隐空间构造非线性决策边界，理论上可表达复杂分类边界。它对参数（如 C、gamma）较敏感，参数选择不当时可能欠拟合或过拟合。 在 EEG 情绪任务中，SVM 常见但需要细致调参，适合与 LR 做“线性 vs 非线性”对照。

mlp 的介绍

MLP 是最基础的前馈神经网络，通过多层线性变换与非线性激活学习特征组合关系。相较传统模型，MLP 具有更高表达能力，但也更依赖训练细节（如标准化、学习率、早停、验证划分）。 在本项目中，MLP 是深度学习最小可运行基线，用来承接从传统机器学习到深度学习的学习路径。

报告页面的功能

报告页面（report.html）承担“结构化实验呈现”的角色。你可以在该页面查看默认基线报告，也可以上传 zip 生成实时实验报告。 页面会给出 subject_dependent 与 subject_independent 两种协议下的 Accuracy 和 Macro-F1，并展示覆盖率信息，帮助你判断当前上传数据是否足以支撑结论； 同时会给出在全体被试数据上训练得到的单一逻辑回归（全量 LR）对你上传子集的预测结果，以及与 save_info 真值的分类差异和分布层面的标量对比（需与导出模型所用特征类型一致，默认 de_lds）。 如果你要写课程报告或论文草稿，这个页面就是主要材料来源。

交互页面的功能

交互页面（demo.html）强调“快速试验与直观观察”。在上传 zip 后，页面会实时解析特征与标签并给出协议指标摘要，同时保留预测分布、 置信度统计与高置信度 trial 的快速检查视图；并与报告页相同，附带全量 LR在上传数据上的指标与混淆矩阵。相比报告页面，它更偏向调试与探索，让你先快速判断上传子集的质量，再回到报告页面做正式解读。

分析流程

分析流程从上传文件开始：系统先从 zip 中读取 `EEF_features` 与 `save_info`，再把 trial 级特征和真实情绪标签按照 session 与行序进行对齐。 对齐后，页面会做两类评估：协议评估使用预先导出的 LR/MLP，按 SEED 设定的 subject_dependent / subject_independent 测试划分汇总指标； 全量单模型评估使用在完整 manifest（如 20 名被试全部 trial）上训练的一个 LR（docs/assets/web_model.json），对你 zip 中每一个「特征与标签同时对齐」的 trial 做预测，并与真值对比（Accuracy、混淆矩阵、真值/预测类别分布的 TVD 与熵等）。 最后，结果会被组织为可直接阅读的实验叙述，形成“指标—解释—结论”的闭环。

上传说明

上传文件必须是一个 zip，并且目录层级固定为 `EEF_features/` 与 `save_info/`。其中 `EEF_features/.mat` 用于提供 trial 特征， `save_info/___save_info.csv` 用于恢复真实情绪标签。为了让 trial 覆盖完整，建议每个被试尽量提供 session 1 到 4 的 save_info 文件。首次上传时浏览器会加载 Pyodide 与 JSZip，因此启动会慢一些；后续再次上传通常会明显更快。
