使用说明

本站点用于 SEED 情绪识别的上传分析与实验结果展示，包含报告页面与交互页面两类功能。

报告页面的功能

  • 入口：report.html
  • 上传 zip 后，输出更像实验报告的结果：subject_dependent 与 subject_independent 两协议的 Accuracy / Macro-F1。
  • 展示覆盖率（valid_trials / expected_trials），用于说明上传数据是否完整。
  • 展示按被试/按fold的明细结果，便于你做分析与汇报。
  • 不上传时，展示默认基线报告与结果表格。

交互页面的功能

  • 入口：demo.html
  • 支持上传 zip 做实时分析（浏览器端解析与推理）。
  • 展示上传子集指标摘要（两协议 + LR/MLP）和覆盖率提示。
  • 保留预测分布、置信度统计、Top trial 等快速观察信息。

分析流程

  1. 准备 zip：`EEF_features/` + `save_info/`。
  2. 上传后解析 `.mat` 特征与 `save_info` 标签，按 trial 对齐真实值与预测值。
  3. 按 SEED 两协议计算 Accuracy / Macro-F1，并给出覆盖率说明。
  4. 在报告页查看结构化结果，在交互页查看可视化与细节。

上传说明

  • zip 结构必须固定：`EEF_features/` + `save_info/`（目录名区分大小写）。
  • `EEF_features/*.mat` 需包含 `de_LDS_1..80`（或 `de_1..80` / `psd_1..80`）这类特征 key。
  • `save_info/*.csv` 每个 session 固定 20 行；情绪从第2列电影路径倒数第二段提取（如 `...\\happy\\xxx.mp4`）。
  • 首次上传时会加载 Pyodide 和 JSZip，等待时间会比后续更长。
